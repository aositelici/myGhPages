再写一些上节课的总结

我们都知道git有三个区域：

  • 1. working directory 工作区
  • 当前目录下除了.git/其他内容就working directory
  • 2. staging area 暂存区
  • 是.git/下的index文件
  • 3. repository 版本库
  • .git/是repository

    • 这三者之间的对应关系可以由下图来表示：



    图中左侧为工作区，右侧为版本库。 .git就是版本库。暂存区是包含在版本库中的一个名为index的文件，index文件中此次暂存的文件的时间戳、长度等信息。有人可能会以后这样的一个虚拟的暂存区有什么意义呢？ 由于这些文件信息的存在，当扫描工作区改动的时候，系统就可以通过扫描这些信息的改变来判断工作区文件是否发生变化。与扫描实际文件内容相比较这样做无疑效率更高。当然这不仅仅是暂存区存在的理由，有了暂存区， 你可以暂存一定数量的文件，当累计到完成一个单位工作量后再进行一次有意义的提交，当然我们提倡小步提交。标记为 "master" 的是 master 分支所代表的目录树。HEAD是一个引用，指向正在使用的分支。
